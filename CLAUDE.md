## Project workflow

Before version work, read `CONTEXT.md`, `plan.md`, and `docs/maintenance.md` in that order. Follow the maintenance guide's priority rules, Definition of Done, Windows validation matrix, and release process. Keep `plan.md` current when scope or status changes; add an ADR for changes to data format, sync semantics, privacy boundaries, or platform architecture.

Never place snippet content, clipboard content, credentials, or private repository addresses in logs, fixtures, issues, or commits. Do not create or push a release tag unless the user explicitly requests a release.
